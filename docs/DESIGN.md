## 一、产品定位

| 项目 | 内容 |
|------|------|
| 产品名称 | 诗词图谱（PoemGraph） |
| 核心目标 | **综合语文素养启蒙**——理解诗句情感 + 了解诗人与历史背景 |
| 目标用户 | 小学生（1–6年级）、教师、家长 |
| 内容定位 | 故事 + 知识点结合，把复杂历史翻译成孩子能懂的语言 |
| 交付形式 | 单一响应式 HTML 文件，支持离线，可微信传阅 |
| 使用场景 | 学生自主探索 / 老师课堂演示（两种模式可切换） |

---

## 二、用户画像

### 小学生（主要用户）
- 年龄：6–12岁，1–6年级
- 需求：好玩、好看、字大、点一点就有反应
- 关键：**只关心自己年级的诗词**，首次打开先选年级，系统记住

### 教师（重要用户）
- 场景：课堂大屏投影、备课参考
- 需求：内容准确、演示流畅、可打印学习单

### 家长（次要用户）
- 场景：陪孩子在家学习
- 需求：内容权威、安全无广告、引导孩子自主探索

## 三、核心功能清单

### F0 — 角色与模式切换

#### 三种角色
| 角色 | 界面特点 | 专属功能 |
|------|---------|---------|
| 👦 学生 | 大字、游戏化、探索感强 | 答题得分、徽章、学习进度 |
| 👨‍👩‍👧 家长 | 陪读模式，内容更完整 | 查看孩子进度、推荐路径 |
| 👩‍🏫 教师 | 课堂演示优先，投影清晰 | 打印学习单、班级演示全屏模式 |

- 右上角"切换角色"按钮，`localStorage` 记住上次选择
- 角色决定：内容深度、界面密度、功能侧重

#### 两种浏览模式（所有角色可切换）
- **探索模式**（默认）：自由点击，无固定顺序，鼓励发现
- **学习模式**：路径引导——读诗 → 注释 → 故事 → 地图 → 做题
- 模式偏好 `localStorage` 记住

### F1 — 年级系统
- 首次打开：全屏选年级（1–6年级）
- `localStorage` 记住，下次直接进入
- 首页只显示该年级及以下诗词
- 右上角随时切换年级

### F2 — 入口（首页并存，导航栏并列）

| 入口 | 说明 |
|------|------|
| 📖 年级诗词 | 按年级分组的诗词列表 |
| 🀄 卡片墙 | 全部诗词卡片，可多维筛选 |
| 🕸 知识图谱 | 交互式力导向关系网络图 |
| 🗺 诗词地图 | 中国 SVG 地图，标注作诗地点 |
| ⏳ 历史时间轴 | 按朝代排列诗词与历史大事 |
| ✍ 诗人长廊 | 诗人维度入口（见 F12） |
| 🚩 闯关 | 学习模式逐课闯关 |

### F12 — 诗人长廊（诗人维度，C 模块）

以"诗人"为主线浏览。三段能力：

- **C1 诗人选择器**：顶部横向一排诗人 chip（圆形印章头像取姓氏字 + 姓名 + 朝代·生年），按「朝代序（唐→宋→元→明→清）+ 生年」排序，可横向滚动，选中项自动滚入视野并高亮墨青。
- **C2 诗人面板**：印章式大头像 + 姓名（楷体大字）+ 朝代·生卒 + 简介 + 统计（入选诗作数 / 足迹地数）+「代表诗作」卡片（点击进详情，展示已得星）+ 足迹/诗作地 mini-map（复用大地图地形+河流图层）。
- **C3 诗人足迹路线**：有游历数据的诗人（李白/杜甫/王维）在面板 mini-map 内画编号①②③虚线路线，并提供「🗺 在大地图看足迹」按钮联动大地图（自动开启该诗人足迹）；无足迹数据的诗人回退为「诗作地点」标注。

### F13 — 时间轴驱动·时空联动（D 模块，参考图灵魂功能）

落于「诗词地图」底部的年份游标，让地图随时间"活"起来：

- **D1 年份游标 + 播放**：底部时间条含 ▶播放/⏸暂停、`1× / 2× / 4×` 调速、可拖动手柄、点击轨道跳转年份。
- **D2 时空联动（核心亮点）**：年份推进时，地图按各诗创作年（`poemYear` 解析模糊年份）**逐步点亮**该年前已问世的诗作地，未到的淡出；顶部实时显示「已问世 N / M 首」。
- **D3 诗人生卒标注**：当选中某诗人足迹时，游标轨道上标注其生/卒年（如李白 生 701 · 卒 762）。
- **D4 朝代分期渐变**：轨道为墨/赭/青系横向渐变色带（**不含红色**），并标注分期（初唐/盛唐/中唐/晚唐·宋·元·明·清）；随朝代筛选自动重定时间范围。
- 实现备忘：播放用 `setInterval`（抗后台标签节流）+ dt 钳制防跳；`poemYear()` 为时间轴与地图共用的年份解析器。

### F14 — 诗作列表联动（E 模块）

诗词地图左侧的列表面板（与地图两栏并排，≤900px 自动堆叠到地图上方）：

- **E1 随年份游标过滤**：列表只显示"截至游标年份已问世"的诗作与大事，顶部计「已问世 N 首」；播放时随年份实时增长。
- **E2 内容筛选 tab**：全部 / 只看诗作 / 只看大事（大事 = 历史事件，与诗作按年份混排）。
- **E3 传播度与代表作滑块**：每首诗人工标注「传播度」（脍炙人口程度，20 首 88–105）；「仅代表作 ⇄ 全部」滑块按传播度阈值过滤，列表项显示传播度数值。
- **E4 列表⇄地图⇄详情联动**：点列表项→地图缩放平移定位到该诗创作地 + 脉冲高亮（必要时推进年份点亮），「读›」按钮→打开详情页。

### F15 — 氛围：朗读与古琴背景音（F 模块）

均坚持「单文件 / 完全离线 / 零额外体积」：

- **F1 朗读（逐句 + 高亮跟读 + 普通话/粤语切换）**：详情页「🔊 朗读」按钮，用浏览器内置 Web Speech API（`speechSynthesis`，**零字节、离线**）。**统一、好听的标准诵读音色**（不按诗人区分），rate 0.84 / pitch 1.0、逐句诵读、句间停顿，当前句在原文区高亮跟读，离开详情页自动停止。**语种切换**：朗读按钮旁「普通话 / 粤语」开关，默认普通话（择优选 zh-CN 标准音如语舒/婷婷，避开趣味/颤音音色），粤语用 zh-HK 善怡；选择存 localStorage `pg_reclang`。〔历史：曾尝试"诗人音色人设"按性格混用不同语种语音，导致李白粤语女声、贺知章颤音、孟浩然女声，经用户反馈废弃。〕
- **F2 古琴背景音（双方案）**：导航「🎵 古琴」面板。
  - 默认 **零字节程序合成**：Web Audio API 以 Karplus-Strong 拨弦算法合成五声音阶（宫商角徵羽）古琴氛围 + 卷积混响，随机疏密排布，可调音量；不增加任何文件体积、纯离线、墨色雅致。
  - **高音质开关**：预留内嵌真实古琴循环 `GUQIN_CLIP`（base64 data URI）；填入可商用授权音频后即用真实音色，留空时自动回退程序合成并提示。
- 视觉/约束：面板与按钮沿用水墨配色（墨青/赭），无红色；音频为可选氛围，默认不自动播放（需用户点击触发，符合浏览器策略）。

### F6 — 学习路径（学习模式专属）
- 路径：读诗原文 → 看注释白话 → 听诗人故事 → 看地图 → 看历史背景 → 做练习
- 每步完成后解锁下一步（进度条）
- 完成全部步骤获得"诗词达人"徽章

### F7 — 诗词地图（手绘古风）
- **风格**：手绘古风绢图感，SVG 内嵌，完全离线
- 色调：旧绢黄底（#f0e8cc）、墨色山形轮廓、毛笔点标记
- 每首诗创作地点标记为**毛笔圆点 + 古地名**
- 悬停/点击标记 → 弹出诗词卡片（含"古地名 → 今XXX"注释）
- 地图区域划分参考**唐代行政区**（大多数诗词为唐代）
- 支持按朝代过滤（显示唐/宋/明清时期地图标记）
- 特色功能：**诗人足迹路线**——选定诗人，在地图上连线他的人生轨迹
- **按诗人展开**：顶部「诗人」横滑选择行（全部 + 22 位）。选中某诗人 → 左侧列表只看其诗 + 顶部诗人简介头（印章头像 / 姓名 / 朝代生卒 / 入选 N 首 / × 关闭）、地图标记只保留其诗作地、有足迹则画①②③。对标参考图"以诗人为主线"。
- **清晰度（缩放保真）**：标记 / 地名 / 足迹 / 名胜 / 山脉名在地图放大时**反向缩放**，保持恒定屏幕尺寸——放大不会变粗变糊、不重叠；描边用 `vector-effect=non-scaling-stroke`。零额外体积。（滚轮缩放需按 ⌘/Ctrl，避免吞掉页面滚动。）

### F8 — 历史时间轴
- 横向时间轴，从秦至清（含重要诗词时代）
- 每首诗标记在对应时间节点
- 点击诗词节点 → 跳转诗词详情
- 历史大事件（安史之乱、宋金南渡等）也标记在轴上

### F9 — 多维搜索
- 搜索维度：诗名、作者、朝代、地点、情感主题
- 实时搜索，前端 JS 实现，无需网络

### F10 — 打印学习单

## 五、视觉设计规范

### 风格定义：水墨画（Ink Wash / 水墨）
以中国传统水墨画为视觉语言——墨色渐变、宣纸质感、大量留白、笔触线条感。
**不使用**饱和色、红色系、彩色按钮。整体克制、雅致、有文人气息。

### 主色调
| 色彩 | 用途 | 色值 | 说明 |
|------|------|------|------|
| 浓墨 | 标题、主文字 | #1c1c1e | 如研磨浓墨 |
| 淡墨 | 副文字、边框 | #5c5c6e | 如第二遍墨 |
| 枯墨 | 辅助信息 | #9a9aaa | 如淡墨晕染 |
| 宣纸白 | 主背景 | #f7f5f0 | 略带暖意的白 |
| 素纸 | 卡片背景 | #efece4 | 旧宣纸色 |
| 淡墨底 | 区块背景 | #e8e5dc | 浅灰绿底 |
| 墨青（点缀） | 选中状态、强调 | #2d5a6b | 青墨，仅用于关键交互 |
| 远山蓝（点缀） | 图谱连线、地图 | #7899a8 | 淡蓝如远山 |
| 竹绿（点缀） | 完成/成功状态 | #5a8a6a | 竹叶绿 |
| 赭石（点缀） | 警示/错误 | #a0603a | 仅用于答题错误 |

> **原则**：点缀色每屏面积占比 < 15%，其余全为墨色 + 纸色。

### 设计原则
1. **留白即设计**：大量空白是水墨美学的核心，避免堆砌内容
2. **墨色层次**：用墨色深浅（而非多色）区分信息层级
3. **笔触质感**：边框用细线（0.5–1px），卡片阴影用墨色低透明度
4. **儿童可读**：正文 ≥ 16px，诗词大字 ≥ 24px，行距 ≥ 1.9
5. **触控友好**：按钮/节点触控区域 ≥ 44px
6. **投影清晰**：主内容高对比度（墨/白），老师课堂投影清晰
7. **装饰元素**：山形轮廓、竹叶、芦苇——用 SVG 简笔描绘，不喧宾夺主

## 七、技术架构

### 技术选型

| 技术 | 用途 | 版本策略 |
|------|------|---------|
| HTML/CSS/JS | 主体架构，无框架 | — |
| Canvas 2D API | 知识图谱力导向图（自写，无CDN） | v1 ✅ |
| Three.js（内联） | 3D知识图谱炫酷模式 | v1 ✅（内联避免CDN） |
| SVG | 中国地图、时间轴、插图 | v1 ✅ |
| localStorage | 年级/进度/偏好/收藏 | v1 ✅ |
| CSS @media print | 打印学习单 | v1 ✅ |

### 文件结构
```
PoemGraph/
├── poemgraph.html     # 主交付文件（单文件，≤3MB）
├── CLAUDE.md          # 项目规则（AI执行规范）
└── docs/
    ├── DESIGN.md      # 产品设计文档（本文件）
    └── PROJECT_PLAN.md # 项目计划与进度
```

### 数据策略
- 全部诗词数据内嵌于 HTML 的 JS 对象
- 用户进度/收藏/偏好：`localStorage`
- 无后端，无账号，无网络依赖

## 八、关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 单文件 vs 多文件 | 单文件 | 微信传阅、双击即开，无安装 |
| 探索 vs 学习模式 | 两种可切换 | 兼顾自主探索与教学引导 |
| 数据库 vs localStorage | localStorage | 无后端，离线，无账号 |
| 2D vs 3D图谱 | 默认2D，可切换3D（Three.js内联） | 2D稳定，3D炫酷满足探索欲 |
| 响应式 vs 双版本 | 一套响应式 | 降低维护成本 |
| CDN vs 内联 | 内联Three.js | 确保离线可用 |
| 20首 vs 更多 | 先做好20首 | 质量优先，可持续扩展 |
