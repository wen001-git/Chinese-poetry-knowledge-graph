# PoemGraph 项目规则

## 跨 AI 工具接手（强制）

本项目会在多个 AI 编程工具（不同 Claude Code 账号 / Codex 等）间切换开发。
- **接手第一步**：先读仓库根 `AGENTS.md`（单一事实来源），再按需读 `docs/PROJECT_PLAN.md`、`docs/DESIGN.md`，据此直接继续，不要全量重扫代码库。
- **收尾**：更新 `AGENTS.md` 的「当前状态 / 下一步 TODO」+ `docs/PROJECT_PLAN.md` 变更日志。
- 项目状态以仓库文件为准（可移植），不依赖任何工具私有记忆。

---

## 文档维护规则（强制执行）

### 1. 项目计划文档 `docs/PROJECT_PLAN.md`
- 每次完成一个功能模块或里程碑，**立即**更新进度状态
- 状态标记：`⬜ 待开始` → `🔄 进行中` → `✅ 已完成`
- 每次更新时在文档末尾的`## 变更日志`中追加一行记录

### 2. 设计文档 `docs/DESIGN.md`
- 每次用户提出新需求、修改需求、或做出架构决策时，**立即**更新设计文档
- 设计文档反映**当前最新**的产品定义，不保留历史草稿
- 每次更新时在文档末尾的`## 变更日志`中追加一行记录

### 3. 触发更新的时机
- 用户确认一个新功能需求 → 更新 DESIGN.md 需求部分 + PROJECT_PLAN.md 计划部分
- 完成一个开发任务 → 更新 PROJECT_PLAN.md 进度
- 用户否定或修改某个设计决策 → 更新 DESIGN.md 对应章节
- 开始新一轮对话前，读取这两个文档以恢复上下文

### 4. 每次对话结束前检查清单
- [ ] PROJECT_PLAN.md 进度是否已更新？
- [ ] DESIGN.md 是否反映了本次对话中的所有决策？
- [ ] 两份文档的变更日志是否已追加？

---

## 里程碑测试规则（强制执行）

- **每完成一个里程碑（M0–M10）的修改后，必须立即做一次全面测试**，不得跳过。
- 测试内容至少覆盖：断网离线可用、五入口与详情页可正常渲染、本里程碑新增/改动功能、控制台无报错、水墨配色无违规（无红色主色）、iPad/桌面响应式。
- 测试使用 Claude Preview / 浏览器 MCP（启动本地服务、截图、读控制台日志），不得让用户手动验证。
- 测试通过后再在 PROJECT_PLAN.md 标记该里程碑为 ✅，并在变更日志记录"已测试"。

## 视觉风格（强制约束）

**风格：水墨画（Ink Wash）**，以下规则不可违反：
- 背景色：宣纸白 `#f7f5f0`，不使用纯白或彩色背景
- 主文字：浓墨 `#1c1c1e`，副文字 `#5c5c6e`
- **禁止使用红色系（#c0392b 等）作为主色或大面积色**
- 强调色仅用墨青 `#2d5a6b`，面积 < 15%
- 图谱节点用墨色系（见 DESIGN.md 第五节）
- 卡片背景：`#efece4`，边框：`1px solid #d8d4c8`
- 阴影：`box-shadow: 0 2px 12px rgba(28,28,30,.08)`，不用彩色阴影
- 装饰元素：水墨山形、竹叶 SVG，不使用彩色插图

## 技术规范

- **交付物**：单一 HTML 文件 `poemgraph.html`，无外部依赖，完全离线可用
- **数据存储**：所有诗词数据内嵌在 HTML 的 JS 对象中；用户进度/偏好用 `localStorage`
- **图谱库**：不使用外部库，用原生 Canvas 自绘力导向图（离线零依赖）
- **3D模式**：Canvas 伪 3D（球形节点透视旋转），不内联 Three.js，控制文件体积
- **响应式**：支持 iPad（768px+）和桌面（1024px+）

## 内容规范

- 诗词数据与**统编版**小学语文课本对齐（1–6年级）
- **内容优先来源**：统编版教材 PDF（GitHub: wen001-git/ChinaTextbook，路径 `小学/语文/统编版`），原文、拼音、字词注释以教材为准
- 诗词数据与人教版小学语文课本对齐（1–6年级）
- 每首诗必须包含：原文+拼音、诗意讲解、诗人故事、历史背景、地理位置、趣味测验（4题）
- 诗意讲解用小学生能懂的语言，按年级调整深度
